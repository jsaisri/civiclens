## Project Description

CivicLens is a full-stack data analytics assistant designed specifically for mission-driven organizations. It allows teams with little to no data engineering experience to upload CSV or Excel files, automatically clean and validate the data, explore dashboards, and use Claude to answer natural-language questions, summarize trends, and produce polished reports for donors, boards, and community stakeholders.

---

## Problem Statement

Nonprofits, food banks, and public-health agencies collect enormous amounts of operational data — meal distributions, client demographics, inventory counts, health outcomes — but rarely have the technical resources to analyze it effectively. Insights get buried in spreadsheets, reports take days to write manually, and leadership teams make decisions without a clear picture of what the data actually says.

CivicLens bridges this gap by combining automated data processing with conversational AI so that program managers, not just data scientists, can unlock the value in their own data.

## Target Users

- **Food bank operations teams** tracking inventory, distribution volumes, and client counts

## Architecture Overview

```
┌─────────────────────────────────────────────────────────┐
│                        Frontend                          │
│         React + Tailwind + Recharts (Port 3000)          │
└───────────────────────┬─────────────────────────────────┘
                        │ HTTP / REST
┌───────────────────────▼─────────────────────────────────┐
│                     FastAPI Backend                       │
│         /api/upload  /api/query  /api/report             │
└──────┬────────────────┬──────────────────┬──────────────┘
       │                │                  │
┌──────▼──────┐  ┌──────▼──────┐  ┌───────▼──────┐
│  Data       │  │  PostgreSQL  │  │  Claude API  │
│  Processing │  │  Database   │  │  (Anthropic) │
│  (Pandas)   │  │             │  │              │
└─────────────┘  └─────────────┘  └──────────────┘
```

**Data flow:**
1. User uploads a file via the frontend
2. Backend receives and passes it to the data processing layer
3. Python scripts clean, validate, and load data into PostgreSQL
4. Dashboard queries the database for charts and KPIs
5. User submits a natural-language question or requests a report
6. Backend builds a structured prompt and calls Claude API
7. Claude's response is returned and rendered in the UI

## Setup Instructions

### Prerequisites
- Python 3.11+
- Node.js 18+
- PostgreSQL 15+
- An [Anthropic API key](https://console.anthropic.com/)

### 1. Clone the repository
```bash
git clone https://github.com/jsaisri/civiclens.git
cd civiclens
```

### 2. Set up environment variables
```bash
cp .env.example .env
# Open .env and fill in your values
```

### 3. Install Python dependencies
```bash
pip install -r requirements.txt
```

### 4. Install frontend dependencies
```bash
cd frontend
npm install
```

### 5. Set up the database
```bash
# Create a PostgreSQL database named civiclens
# Then run migrations:
cd database
alembic upgrade head
```

### 6. Run the backend
```bash
cd backend
uvicorn api.main:app --reload --port 8000
```

### 7. Run the frontend
```bash
cd frontend
npm run dev
```

Visit `http://localhost:3000` in your browser.

## Claude Integration Plan

CivicLens uses Claude as the reasoning engine for three core workflows:

### 1. Natural Language Q&A
The user types a question like _"Which zip codes had the highest food distribution in March?"_. The backend:
- Inspects the database schema
- Builds a prompt that includes schema context + the question
- Calls Claude to generate a SQL query
- Executes the query and returns formatted results

### 2. Dataset Summaries
After a file is uploaded and processed, the user can click "Summarize." The backend:
- Computes descriptive statistics (row count, date range, key columns, null rates)
- Passes this context to Claude with a summary prompt
- Returns a 2–3 paragraph plain-English narrative

### 3. Stakeholder Reports
The user selects a date range and report type (board update, donor report, grant narrative). The backend:
- Queries relevant data
- Builds a structured prompt with organization context, data stats, and tone instructions
- Claude drafts the full report
- Output is rendered in the UI and downloadable as PDF or Word

All prompts are stored in `/claude_prompts/` as editable Markdown templates, making them easy to customize per organization.

## Dashboard Plan

The dashboard will display the following for each uploaded dataset:

- **Summary KPI cards** — total records, date range, key metric totals
- **Time-series line chart** — trends over weeks/months
- **Bar chart** — breakdown by category (location, program type, etc.)
- **Data quality panel** — missing value counts, duplicate rows, flagged anomalies
- **Recent activity feed** — last uploads, last Claude queries

Each chart component will live in `/dashboard/charts/` as a reusable React component.
